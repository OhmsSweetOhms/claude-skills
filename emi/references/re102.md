## Measurement Meaning

An antenna-based RE102 result is field strength:

```text
dBuV/m = dBm + 106.9897 + antenna_factor_dB_per_m + cable_loss_dB - external_gain_dB
```

If `--equivalent-distance-m` is used, the project also applies:

```text
20*log10(measured_distance_m / equivalent_distance_m)
```

Use that only when intentionally normalizing between measurement distances.

For cabled signal-generator checks, bypass antenna factors. Those runs are
receiver input power:

```text
dBuV_50ohm = dBm + 106.9897
```

## Live Scan Pattern

Use a measurement-set directory, not the old flat `data/re102` output, for new
work. See `references/data-layout.md`.

Typical noise-floor scan:

```bash
.venv/bin/python tools/emi_control.py re102 scan <test_id>_noise_floor_horizontal \
  --host 192.168.0.101 \
  --backend @py \
  --antenna antennas/TBMA1B_gain_antenna_factor.json \
  --distance-m 1 \
  --polarization horizontal \
  --cable-loss-db 1.5 \
  --out-dir data/re102/measurements/<test_id>/runs/001_noise_floor_horizontal/raw \
  --out-json data/re102/measurements/<test_id>/runs/001_noise_floor_horizontal/derived/field_strength.json
```

Typical UUT-on scan:

```bash
.venv/bin/python tools/emi_control.py re102 scan <test_id>_uut_on_horizontal \
  --host 192.168.0.101 \
  --backend @py \
  --antenna antennas/TBMA1B_gain_antenna_factor.json \
  --distance-m 1 \
  --polarization horizontal \
  --cable-loss-db 1.5 \
  --out-dir data/re102/measurements/<test_id>/runs/002_uut_on_horizontal/raw \
  --out-json data/re102/measurements/<test_id>/runs/002_uut_on_horizontal/derived/field_strength.json
```

Current CLI behavior writes raw CSVs to `--out-dir` and writes the corrected
field-strength JSON to `--out-json` when provided. For the preferred layout,
always pass `--out-json` into the run's `derived/` directory.

## Plot And Report Pattern

An actual RE102-style plot should compare:

- Noise floor / ambient, UUT off.
- UUT on.
- Selected limit line.

Generate a report SVG:

```bash
.venv/bin/python tools/emi_control.py re102 report-svg \
  data/re102/measurements/<test_id>/runs/001_noise_floor_horizontal/derived/field_strength.json \
  data/re102/measurements/<test_id>/runs/002_uut_on_horizontal/derived/field_strength.json \
  --label 'Noise floor, UUT off' \
  --label 'UUT on' \
  --limit aircraft_fixed_wing_internal_lt25m \
  --out data/re102/measurements/<test_id>/reports/<test_id>_re102_fixed_wing_internal_lt25m.svg
```

Report expectations:

- One full-range overview over the antenna range.
- Failure/detail panels using spans that match antenna-factor scan segments.
- Raw/corrected data should remain available for replotting.
