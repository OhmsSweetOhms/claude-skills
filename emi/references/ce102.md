# CE102 Reference

CE102 is the conducted-emissions workflow for power leads. The EMI project has
first-pass CE102 tooling based on the Tekbox TBL5016-1 LISN calibration data,
the Rigol RSA5065 trace-export path, and the Siglent SDG2042X signal generator.

## Measurement Meaning

CE102 is measured as voltage, not field strength:

```text
dBuV
```

No antenna factor is applied.

For a 50 ohm analyzer reading in dBm:

```text
dBuV_50ohm = dBm + 106.9897
```

For the current expected chain:

```text
UUT power lead -> TBL5016-1 50 uH LISN -> 20 dB attenuator -> cable -> RSA input
```

The first-pass correction should be:

```text
dBuV_at_DUT_terminals =
    dBm + 106.9897
  + attenuator_loss_dB
  + cable_loss_dB
  + LISN_correction_dB
  - external_gain_dB
```

The EMI project uses the TBL5016-1 voltage-division-ratio table as the LISN
correction:

```text
LISN_correction_dB = -voltage_division_ratio_DUT_to_RF_port_dB
```

The inline attenuator defaults to 20 dB.

## Active Hardware Decisions

- LISN: Tekbox TBL5016-1.
- Active LISN configuration: `50uh` only (`50 ohm // 50 uH`).
- Do not suggest or select `50uh_5ohm` unless the user explicitly says the
  hardware has been changed and the CLI has been updated to allow it.
- The TBL5016-1 `50uh` voltage-division table starts at 15 kHz. Normal CE102
  data scans clip to 15 kHz through 10 MHz for defensible table-based
  corrections.
- MIL-STD-461F still requires a 10 kHz measurement-system check. For that one
  low-frequency check point, the project uses the MIL-STD-461F 50 uH
  coupling-capacitor correction estimate, not the Tekbox table.

## Information To Capture Before Running

- LISN serial number if available.
- Power type and line under test: DC, AC, positive, negative, line, neutral.
- Confirm physical LISN configuration is still `50uh`.
- Confirm source voltage for CE102 limit selection: `basic_28v`, `115v`,
  `220v`, `270v`, or `ge440v`.
- Cable loss between LISN/attenuator/RSA.
- RSA input protection requirements.
- Whether this is generator-only bench verification, a LISN system check, a
  noise-floor scan, or a UUT-on scan.

## Plot Pattern

Use the same reporting idea as RE102:

- Noise floor / ambient, UUT off.
- UUT on.
- Selected CE102 limit line.
- Full-range overview plus failure/detail panels.

The y-axis is `dBuV`, not `dBuV/m`.

## Measurement-System Check

MIL-STD-461F 5.5.3.4 requires sine-wave injections at 10 kHz, 100 kHz, 2 MHz,
and 10 MHz, at least 6 dB below the applicable CE102 limit. The corrected
receiver reading must be within +/-3 dB of the injected level.

For the `basic_28v` limit and default 6 dB margin:

```text
10 kHz    88 dBuV   25.1189 mVrms   71.0469 mVpp   MIL 50 uH correction estimate 4.453 dB
100 kHz   68 dBuV    2.5119 mVrms    7.1047 mVpp   TBL5016-1 50uh table correction 1.190 dB
2 MHz     54 dBuV  501.187 uVrms     1.4176 mVpp   TBL5016-1 50uh table correction 0.020 dB
10 MHz    54 dBuV  501.187 uVrms     1.4176 mVpp   TBL5016-1 50uh table correction 0.040 dB
```

Signal generator:

- Siglent SDG2042X observed at `192.168.0.2`.
- IDN observed: `Siglent Technologies,SDG2042X,SD\XCAC6R088,2.01.01.35R3B2`.
- SCPI port: 5025.
- Use channel C1 unless the user specifies otherwise.
- Set the SDG output load to 50 ohm. When checking directly on a high-Z scope
  without a 50 ohm terminator, expect about twice the programmed Vpp.
- On 2026-05-05, generator-only scope checks passed for all four points. The
  actual LISN system check still needs to be run after moving locations and
  connecting the LISN. The SDG output was turned off after the bench check.

## Direct SDG-To-RSA Smoke Test

Use this only to verify bench plumbing before the LISN is available or before
moving locations. It validates SDG control, RSA scan/export, FTP retrieval, CSV
parsing, CE102 correction plumbing, limit summaries, and SVG plotting. It does
not validate LISN behavior or replace the MIL-STD-461F measurement-system
check.

Physical setup used on 2026-05-05:

```text
SDG C1 -> 20 dB attenuator -> RSA RF input
```

Recommended smoke point:

```text
100 kHz system-check tone
SDG level: 68 dBuV = 2.5119 mVrms = 7.1047 mVpp into 50 ohm
Expected RSA input after 20 dB attenuator: about -58.99 dBm
```

Commands:

```bash
.venv/bin/python tools/emi_control.py ce102 system-check-tone 100000 \
  --siggen-host 192.168.0.2 \
  --limit basic_28v \
  --apply

.venv/bin/python tools/emi_control.py ce102 system-check-scan 100000 \
  --name ce102_direct_sdg_100k_narrow \
  --host 192.168.0.101 \
  --attenuator-loss-db 20 \
  --line direct_sdg \
  --limit basic_28v \
  --out-dir data/ce102/smoke/direct_sdg_100k_narrow/raw \
  --out-json data/ce102/smoke/direct_sdg_100k_narrow/derived/conducted_voltage.json

.venv/bin/python tools/emi_control.py ce102 scan ce102_direct_sdg_100k_full \
  --host 192.168.0.101 \
  --line direct_sdg \
  --attenuator-loss-db 20 \
  --out-dir data/ce102/smoke/direct_sdg_100k_full/raw \
  --out-json data/ce102/smoke/direct_sdg_100k_full/derived/conducted_voltage.json
```

Observed result on 2026-05-05:

```text
Narrow scan nearest 100 kHz bin: -58.755 dBm, +0.235 dB from expected
Narrow corrected result: 69.425 dBuV, +1.425 dB from 68 dBuV target, PASS
Full scan peak: 100.050 kHz, -58.717 dBm raw, 69.462 dBuV corrected
Full scan limit margin: -4.533 dB below basic_28v
```

The corrected direct-smoke value is intentionally about 1.2 dB above the SDG
injected value because the CE102 path still applies the TBL5016-1 `50uh`
100 kHz LISN table correction even though the LISN is not physically present.
That is acceptable for this smoke test; use the raw dBm check to judge the
direct RF level.

Artifacts from the passing smoke test:

```text

data/ce102/smoke/direct_sdg_100k_full/plots/ce102_direct_sdg_100k_full.svg
```

The SDG output was turned off after the smoke test:


## Local Commands

Preview default TBL5016-1 scan. The manual says the default hardware
configuration is `50 ohm // 50 uH`; its voltage-division table starts at
15 kHz, so the default plan clips to 15 kHz through 10 MHz:

```bash
.venv/bin/python tools/emi_control.py ce102 plan --show 0
```

Run a live scan:

```bash
.venv/bin/python tools/emi_control.py ce102 scan ce102_noise_floor_positive \
  --host 192.168.0.101 \
  --line positive \
  --attenuator-loss-db 20 \
  --cable-loss-db 0 \
  --out-dir data/ce102/measurements/<test_id>/runs/001_noise_floor_positive/raw \
  --out-json data/ce102/measurements/<test_id>/runs/001_noise_floor_positive/derived/conducted_voltage.json
```

## Caution

Do not use antenna factors for CE102. Shared pieces are raw RSA CSV export and
parsing, plotting primitives, limit-line handling, and manifest/report
organization.
