# Standard Project Structure

Every SoC module project follows this layout. Create it at the start of Stage 0.

---

## Directory layout

```
project_name/
├── src/               # VHDL source files (synthesisable RTL)
├── tb/                # Python TB, SV TB, DPI-C bridge
├── sw/                # C/C++ bare-metal drivers (header + source)
├── build/
│   ├── sim/           # Simulation scripts + Xsim artifacts
│   ├── synth/         # Synthesis TCL scripts + Vivado reports
│   ├── ip/            # IP packaging artifacts (generated, gitignored)
│   ├── py/            # Python outputs (plots, generated data)
│   ├── logs/          # Pipeline run charts + transition logs (auto-generated)
│   └── artifacts/     # Claude scratch space (temp files, intermediate outputs)
├── docs/              # Architecture diagrams (Mermaid source + rendered PNGs)
├── socks.json         # Project config (name, scope, board, sub-designs)
├── CLAUDE.md          # Project guide (generated by Stage 11)
└── .gitignore         # Standard ignores for Vivado/Xsim artifacts
```

### `src/` — VHDL sources

One file per entity. Core module + any sub-modules. No testbench code.

### `tb/` — Testbenches and audit

- `module_tb.py` — Cycle-accurate Python model + tests
- `module_tb.sv` — SystemVerilog Xsim testbench
- `synthesis_audit.py` — Static VHDL checks (12 rules)
- `module_vcd_verify.py` — VCD post-simulation verifier (optional)

### `sw/` — Bare-metal drivers

- `module_name.h` — Register defines, bit fields, driver struct, prototypes
- `module_name.c` — Implementation with usage example

### `docs/` — Architecture diagrams

- `ARCHITECTURE.md` — Mermaid source (Data Flow + Clocking diagrams, Rate Summary tables)
- `ARCHITECTURE_dataflow.png` — Rendered data flow diagram
- `ARCHITECTURE_clocking.png` — Rendered clocking diagram
- `README.md` — Feature spec, entity interface, build & test, synthesis results

Render with: `cd docs && mmdc -i ARCHITECTURE.md -o arch.png -w 1400 -e png -b white && mv arch-1.png ARCHITECTURE_dataflow.png && mv arch-2.png ARCHITECTURE_clocking.png`

See `references/architecture-diagrams.md` for diagram conventions.

### `build/sim/` — Simulation

- `run_sim.sh` — Compile, simulate, verify (Xsim)
- `dump_signals.tcl` — VCD signal selection (optional)
- All Xsim artifacts (VCD, WDB, logs) land here

### `build/synth/` — Synthesis

- `synth_check.tcl` — Vivado synthesis for utilization + DRC
- `synth_timing.tcl` — Vivado synthesis with timing constraints
- All Vivado reports land here

### `build/py/` — Python outputs

Waveform plots, generated data files, and other Python script outputs.
Scripts should resolve this path from `__file__`:
```python
_PY_DIR = os.path.join(os.path.dirname(os.path.abspath(__file__)), '..', 'build', 'py')
os.makedirs(_PY_DIR, exist_ok=True)
```

### `build/state/` — Project state

Contains `project.json`, the single source of truth for pipeline state: stage
results, input hashes, and next-action suggestions. Created automatically by
workflow commands. See `references/session.md` for the full schema.

### `build/logs/` — Pipeline logs (legacy)

Legacy pipeline logs from older SOCKS versions. New projects use
`build/state/project.json` instead. Migrate with `--migrate`.

### `build/ip/` — IP packaging

Generated by Stage 21 (`ip_package.py`). Contains `package_ip.tcl` (generated
Tcl), `component.xml` (IP-XACT descriptor), `xgui/` (Vivado GUI files),
Vivado logs, and `.ip_hash` (source hash for skip logic). Entirely gitignored
— everything is regenerated from `socks.json` + VHDL sources.

See `references/ip-packaging.md` for the full `ip` schema and interface
detection rules.

### `build/artifacts/` — Scratch space

Claude's working area for intermediate outputs, temp files, generated scripts,
or any other artifacts that don't belong in the source tree. Entirely gitignored.

### Path conventions for scripts in `build/`

Scripts in `build/sim/` and `build/synth/` are two directory levels below the
project root. Use these patterns to find the project root:

**TCL** (synth scripts):
```tcl
set script_dir [file dirname [file normalize [info script]]]
set proj_dir   [file dirname [file dirname $script_dir]]
```

**Bash** (sim scripts):
```bash
SIM_DIR="$(cd "$(dirname "$0")" && pwd)"
PROJ_DIR="$(cd "${SIM_DIR}/../.." && pwd)"
```

## .gitignore template

```gitignore
# Build outputs (tracked scripts survive — gitignore only affects untracked)
build/

# Claude local settings
.claude/
```

Add project-specific lines only if needed (e.g. `*.pdf`, `*.zip`).

`PYTHONPYCACHEPREFIX` is set in build scripts and `socks.py` to redirect
`__pycache__/` into `build/py/`, so no separate `__pycache__` gitignore
line is needed.

All Vivado/Xsim artifacts, synthesis reports, generated TCL, CSV logs,
and Python-generated plots land under `build/` so a single rule covers
everything. Tracked files (shell scripts, hand-written TCL) are
unaffected — gitignore only applies to untracked files.
